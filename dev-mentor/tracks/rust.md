## Phase 2 Rust 分支流程

### 步骤 2.1 — Rust 基础概念

1. **所有权（Ownership）**
   - 解释"每个值只有一个主人"的规则——"Rust 里每个值都有一个'所有者'变量，当所有者离开作用域，值就被自动释放，不需要垃圾回收"
   - 所有权转移（move 语义）——"把值赋给另一个变量后，原变量就不能再用了，就像把书给了别人自己就不能再看了"
   - 克隆（clone）——"如果你真的需要两份，显式调用 .clone()，Rust 不允许隐式复制"

2. **借用（Borrowing）**
   - 不可变引用 `&T`——"借来看看，不能改"
   - 可变引用 `&mut T`——"借来改改，但同一时间只能有一个人改"
   - 借用规则——"多个不可变引用 OR 一个可变引用，不能同时存在"
   - 用前端类比——"就像前端的状态管理：不可变引用像只读 state，可变引用像 setState"

3. **生命周期（Lifetime）**
   - 解释编译器如何追踪引用的有效性——"生命周期是编译器的'保质期标签'，确保引用不会活过它指向的数据"
   - 大多数情况编译器自动推断，无需手写
   - 需要显式标注的场景：函数返回引用、结构体持有引用

4. **模式匹配（Pattern Matching）**
   - match 表达式——"类似前端的 switch 但更强大，必须穷举所有可能"
   - if let / while let——"只关心某一种情况时用"
   - 解构——"从复杂类型中提取值"

⏸ **暂停点**：确认用户理解所有权和借用后再继续。这是 Rust 最核心也最独特的概念。如果用户困惑，用更多类比或小型代码练习巩固。

---

### 步骤 2.2 — 类型系统深入

1. **struct / enum / trait**
   - struct——"类似 TypeScript 的 interface，定义数据结构"
   - enum——"类似 TypeScript 的 union type，但每个变体可以携带不同数据"
   - trait——"类似 TypeScript 的 interface + 默认实现，定义共享行为"
   - impl 块——"给类型实现方法，就像给类写方法"

2. **泛型（Generics）**
   - 泛型函数和泛型结构体——"和 TypeScript 的泛型一样，但 Rust 在编译时单态化，零运行时开销"
   - trait bounds——"约束泛型必须实现哪些能力"
   - 泛型 vs 具体类型的选择——"需要灵活用泛型，确定类型直接写"

3. **错误处理：Result<T, E> / Option<T>**
   - Option<T>——"可能没有值，类似 TypeScript 的 T | null 但必须显式处理"
   - Result<T, E>——"可能出错，类似 Promise 的 resolve/reject 但更严格，必须在编译时处理"
   - ? 运算符——"错误处理的快捷方式，遇到错误自动向上传播"
   - thiserror + anyhow 实践——"库代码用 thiserror 定义错误类型，应用代码用 anyhow 简化处理"

⏸ **暂停点**：确认用户理解类型系统和错误处理模式后再继续。建议写几个小练习（如实现一个返回 Result 的函数）。

### 步骤 2.3 — Cargo 与项目结构

1. **Cargo.toml 配置**
   - [dependencies] 依赖管理——"类似 package.json 的 dependencies"
   - [dev-dependencies] 开发依赖——"只在测试时用"
   - feature flags——"条件编译，按需启用功能"
   - workspace——"一个仓库多个 crate，类似 monorepo"

2. **常用命令**
   - `cargo build` / `cargo build --release`——"debug 模式快速编译，release 模式优化性能"
   - `cargo run`——"编译并运行"
   - `cargo test`——"运行所有测试"
   - `cargo clippy`——"Rust 的 linter，帮你发现代码问题"
   - `cargo fmt`——"自动格式化代码"
   - `cargo doc --open`——"生成并打开文档"

3. **项目结构约定**
   - src/main.rs / src/lib.rs——"入口文件"
   - src/bin/——"多个可执行文件"
   - tests/——"集成测试目录"
   - modules（mod）——"代码组织和模块化"

⏸ **暂停点**：确认用户能独立使用 Cargo 创建项目、添加依赖、运行测试后再继续。

---

### 步骤 2.4 — 项目实战

1. **从小功能开始，逐步增加复杂度**
   - 先实现一个最小可用功能——"先让一个最简单的版本跑起来"
   - 每次只加一个功能——"小步快跑，每步都能编译通过"

2. **引入第三方 crate**
   - Serde——"序列化/反序列化，几乎所有 Rust 项目都需要"
   - Tokio——"异步运行时，网络编程的基础"
   - 根据项目类型选择其他 crate（见技术栈表）

3. **每个功能的开发流程**
   - 讲解思路——"接下来要做 X，思路是 Y"
   - 给代码
   - 解释代码中的关键点
   - 等用户跑通后继续

⏸ **暂停点**：每完成一个功能模块后暂停。Rust 编译错误可能比较难懂，给足时间让用户理解和调试。

## 理想结果

学完 Rust 方向后，用户应该能够：

- ✅ 理解所有权、借用、生命周期（Rust 最核心的概念）
- ✅ 熟练使用类型系统（struct/enum/trait/generics）
- ✅ 掌握 Rust 错误处理模式（Result/Option/?运算符）
- ✅ 使用 Cargo 管理项目和依赖
- ✅ 用 Rust 构建一个完整的 CLI 或 Web 服务
- ✅ 理解 Rust 独有的工程哲学（零成本抽象、编译时安全）
