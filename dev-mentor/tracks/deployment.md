## Phase 2 部署分支流程

### 步骤 2.1 — Linux 基础

1. **常用命令**
   - 文件操作：ls / cd / cp / mv / rm / mkdir / chmod
   - 文件查看：cat / less / head / tail / grep / find
   - 系统信息：top / htop / df / free / uname
   - 网络工具：curl / wget / netstat / ss / ping
   - 服务管理：systemctl start/stop/status/restart

2. **文件权限与用户管理**
   - 权限模型：rwx（读/写/执行）——"就像文件夹的访问权限：能看、能改、能运行"
   - chmod / chown 命令
   - 用户和用户组
   - sudo 的原理——"临时获得管理员权限，不是真的变成 root"

3. **SSH 连接与安全配置**
   - SSH 密钥登录配置——"密钥登录比密码安全得多，就像门禁卡比钥匙安全"
   - 禁止密码登录（安全加固）
   - SSH config 文件简化连接
   - 常用 SSH 隧道和端口转发

⏸ **暂停点**：确认用户能通过 SSH 连接服务器、执行基本命令后再继续。如果用户已有 Linux 经验，快速跳过。

---

### 步骤 2.2 — Docker 容器化

1. **Docker 核心概念**
   - 镜像（Image）——"镜像就是应用的打包模板，包含代码和所有依赖，像集装箱"
   - 容器（Container）——"容器是镜像的运行实例，就像用模板创建的应用副本"
   - 卷（Volume）——"卷是容器的持久化存储，容器删了数据还在"
   - 网络（Network）——"容器之间通信的虚拟网络"

2. **编写 Dockerfile**
   - 逐行解释每个指令——"FROM 选基础镜像、WORKDIR 设工作目录、COPY 复制代码、RUN 执行命令、EXPOSE 声明端口、CMD 启动应用"
   - 多阶段构建——"构建阶段和运行阶段分开，最终镜像更小更安全"
   - .dockerignore——"忽略不需要的文件，加速构建、减小镜像体积"

3. **docker-compose 多服务编排**
   - 编写 docker-compose.yml——"把多个容器的配置写在一个文件里，一键启动整个应用栈"
   - 常见编排：app + 数据库 + Redis
   - 环境变量管理——".env 文件存放配置，不要把密钥写死在 compose 文件里"
   - 常用命令：up / down / logs / exec / ps

⏸ **暂停点**：确认用户能用 Docker 成功构建镜像、启动容器后再继续。让用户实际操作一遍。

---

### 步骤 2.3 — Nginx 与 HTTPS

1. **Nginx 安装与反向代理配置**
   - 解释什么是 Nginx——"Nginx 是互联网的'前台接待'，接收请求然后分发给对应的服务"
   - 反向代理 vs 正向代理——"反向代理是服务器端的代理（用户不知道背后是谁），正向代理是客户端的代理（翻墙就是正向代理）"
   - 配置反向代理——"把 80 端口的请求转发到应用容器的端口"
   - 负载均衡基础——"多个应用实例分摊流量"

2. **静态资源托管**
   - 配置静态文件服务
   - Gzip 压缩——"压缩响应体积，加速传输"
   - 缓存策略——"静态资源设置长期缓存，加速重复访问"

3. **SSL/TLS 与 HTTPS 配置**
   - 解释什么是 HTTPS——"HTTP 是明文传输，HTTPS 是加密传输，防止数据被窃听和篡改"
   - SSL/TLS 证书——"证书就像网站的身份证，由权威机构颁发，证明你是你"
   - Let's Encrypt + Certbot——"免费的证书颁发服务，自动申请和续期"
   - 自动续期配置

⏸ **暂停点**：确认用户配置好 Nginx 反向代理和 HTTPS 后再继续。这是上线前的关键环节。

---

### 步骤 2.4 — 日志、监控与 CI/CD（可选进阶）

> 此步骤为部署方向的可选进阶内容，根据用户进度和需求决定是否深入。

1. **日志配置与查看**
   - 日志分级（debug / info / warn / error）
   - 应用日志配置
   - Docker / Nginx 日志查看
   - 常见排查场景：502 怎么查、接口超时怎么查

2. **基础监控**
   - Uptime Kuma 部署与配置
   - 告警通知配置
   - 数据库自动备份——"定时脚本 + cron，每天备份数据库，这是最后的保险"

3. **CI/CD 基础**
   - 解释什么是 CI/CD——"CI 是持续集成（自动测试），CD 是持续部署（自动上线），push 代码就能自动完成一切"
   - GitHub Actions 配置
   - 自动化流程：push → test → build → deploy

⏸ **暂停点**：每完成一个子主题后暂停。日志和监控是长期运维的基础，CI/CD 可以在项目更成熟后再配置。

- ✅ 配置 Nginx 反向代理和 HTTPS
- ✅ 理解 DNS 和域名解析
- ✅ 配置日志和基础监控
