## 推荐技术栈（版本锚定）

| 类别 | 推荐技术 | 版本要求 | 说明 |
|---|---|---|---|
| **语言（前端转后端）** | TypeScript | ≥ 5.x（使用最新稳定版） | 语法熟悉，转型成本低 |
| **语言（追求性能）** | Go | ≥ 1.22（使用最新稳定版） | 轻量高性能，部署简单 |
| **语言（快速原型）** | Python | ≥ 3.12（使用最新稳定版） | 生态丰富，开发速度快 |
| **框架（TS）** | NestJS | ≥ 10.x（使用最新稳定版） | 企业级，与 Angular 概念相通 |
| **框架（Go）** | Gin | ≥ 1.9（使用最新稳定版） | 轻量高性能 |
| **框架（Python）** | FastAPI | ≥ 0.110（使用最新稳定版） | 异步、自动文档 |
| **数据库** | PostgreSQL | ≥ 16（使用最新稳定版） | 类型安全、功能全面 |
| **数据库（灵活 schema）** | MongoDB | ≥ 7.0（使用最新稳定版） | 文档型，灵活 schema |
| **缓存** | Redis | ≥ 7.2（使用最新稳定版） | 内存缓存，高性能 |
| **ORM（TS）** | Prisma | ≥ 5.x（使用最新稳定版） | 类型安全首选 |
| **ORM（Go）** | GORM | ≥ 1.25（使用最新稳定版） | Go 生态主流 ORM |
| **ORM（Python）** | SQLAlchemy | ≥ 2.x（使用最新稳定版） | Python 生态成熟 ORM |

> 以上为推荐默认值。如果用户有特定偏好，尊重用户选择。版本号标注最低推荐版本，建议始终使用最新稳定版。

## Phase 2 后端分支流程

### 步骤 2.1 — 数据库连接与 ORM 配置

1. 解释什么是 ORM（Object-Relational Mapping，对象关系映射）——"ORM 就是把数据库的表映射成代码里的对象，让你用写代码的方式操作数据库，不用手写 SQL"
2. 解释为什么需要 ORM——"手写 SQL 容易出错、难以维护，ORM 提供类型安全、自动迁移、查询构建"
3. 引导配置数据库连接
4. 创建第一个 Model
5. 运行数据库迁移

⏸ **暂停点**：用户确认数据库连接成功、Model 创建完成后再继续。如有报错，优先教学式排查。

---

### 步骤 2.2 — API 开发

1. 解释 RESTful API 的概念——"REST 是一种设计风格，用 HTTP 方法（GET/POST/PUT/DELETE）对应增删改查操作"
2. 实现第一个 GET 接口（列表查询）
3. 实现第一个 POST 接口（创建数据）
4. 逐步实现 PUT / DELETE 接口
5. 错误处理和响应格式统一——"为什么统一响应格式？因为前端需要一个稳定的数据结构来处理成功和失败"

⏸ **暂停点**：用户确认 CRUD 接口全部跑通后再继续。让用户用 Postman / Thunder Client 逐个测试。

---

### 步骤 2.3 — 认证鉴权

1. 解释认证（Authentication）vs 授权（Authorization）——"认证是'你是谁'，授权是'你能做什么'"
2. 解释 JWT（JSON Web Token）——"JWT 是一个加密的令牌，用户登录后服务器发给他，之后每次请求带上这个令牌，服务器就知道是谁了"
3. 解释 Session 方案——"Session 是服务器端记住用户状态，存在内存或 Redis 中"
4. 逐步实现注册接口
5. 逐步实现登录接口
6. 逐步实现 Token 校验中间件

⏸ **暂停点**：用户确认注册/登录流程跑通、Token 校验生效后再继续。这是一个关键安全环节。

---

### 步骤 2.4 — 业务逻辑开发

1. 按模块逐步实现功能——"我们先把核心业务拆成几个模块，一个一个来"
2. 每个模块的流程：讲解思路 → 给代码 → 解释代码 → 等用户跑通
3. 复杂逻辑拆分为 Service 层——"为什么要分层？Controller 处理请求，Service 处理业务逻辑，Repository 处理数据存取"
4. 添加输入验证（Validation）——"永远不要信任用户输入"

⏸ **暂停点**：每完成一个业务模块后暂停。用户可以在任意模块后提问、休息，或直接进入 Phase 3。

---

## 理想结果

学完后端方向后，用户应该能够：

- ✅ 理解后端核心概念（API、数据库、认证、中间件、分层架构）
- ✅ 独立使用现代框架开发后端服务
- ✅ 设计和实现 RESTful API
- ✅ 配置认证鉴权（JWT / Session）
- ✅ 完成业务逻辑的分层开发
