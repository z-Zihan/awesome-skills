# base: ux — 用户体验基础 / UX Foundation

所有 Skill 类型共享的用户体验评审基础。

## 核心问题 / Core Question

使用者（人和 AI）用起来顺畅吗？

## 二级观察项 / Secondary Observations

- **Information Density / 信息密度** — 信息量是否合理，不轰炸也不贫乏
- **Interaction Pacing / 交互节奏** — 暂停点是否合理，用户是否疲劳

## 评审要点 / Review Points

- 单次输出信息量是否合理（过多=疲劳，过少=来回问）
- 是否有适当的暂停点让用户介入决策
- 报告/输出格式是否清晰可读（表格 > 大段文字）
- 是否有明确的下一步指引（用户看完知道该做什么）
- 触发词是否覆盖常见表达

## 评分锚点 / Scoring Anchors (UX 维度内)

评分锚点的完整定义见 `scoring/SKILL.md` 中的"评分锚点"章节。

综合锚点：
- 12 分=信息密度合理，暂停点恰当，重点突出
- 7 分=信息略多但可接受，交互基本顺畅
- 3 分=信息轰炸或过度简略
