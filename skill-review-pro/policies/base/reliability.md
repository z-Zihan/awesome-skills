# base: reliability — 可靠性基础 / Reliability Foundation

所有 Skill 类型共享的可靠性评审基础。

## 核心问题 / Core Question

这个 Skill 能不能稳定、正确地完成任务？

## 二级观察项 / Secondary Observations

- **Positioning Clarity / 定位清晰度** — 能否快速理解 Skill 干什么、不做什么
- **Instruction Clarity / 指令明确性** — 指令有无歧义、矛盾、缺失
- **Boundary Rationality / 边界合理性** — 职责是否聚焦，有无膨胀

## 评审要点 / Review Points

- 5 秒内能否理解 Skill 的输入/输出/边界
- 核心流程是否有明确的步骤说明
- 是否有"不做什么"的边界定义
- 指令是否存在矛盾（如"严格按模板"和"自由发挥"并存）
- 模糊指令（如"适当处理"、"合理调整"）是否给出了判断标准

## 评分锚点 / Scoring Anchors (Reliability 维度内)

- 定位清晰：10 分制，10=5 秒理解，5=能理解但边界模糊，2=读完不知道干啥
- 指令明确：10 分制，10=无歧义可执行，5=大部分明确，2=多处矛盾
- 边界合理：8 分制，8=聚焦且有"不做什么"，5=基本清晰但略有膨胀，2=明显膨胀
